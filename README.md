# Campus Wastewater Health Alert Dashboard

An AI-assisted wastewater surveillance dashboard for early outbreak detection, built on your **real campus layout** (the map you provided — Hostels B8–B26, Academic Blocks A9–A18, Peepal/Oak/Alder/Pine/Tulasi Mess, Library, Sports Complex, Health Center, etc.).

> **Data note:** There is no live sensor feed connected to this project. Every pH / viral load / bacterial load / turbidity reading is generated by a realistic simulation engine (`frontend/src/data/generateData.js`) seeded on your real building names and campus zones. Swap in real lab CSV exports later by replacing that one file — everything downstream (map, predictions, charts, alerts, chatbot) already reads from a single data shape, so nothing else needs to change.

## Tech stack
- **Frontend:** React + Vite, Tailwind CSS, Framer Motion, Recharts, React Icons
- **Backend:** Node.js + Express (serves the dataset as a REST API — optional, frontend works standalone too)
- **Map:** Your actual uploaded campus map image with interactive clickable hotspots (no external tile server / API key needed)

## Setup

### 1. Frontend (this is all you need to see the full app)
```bash
cd frontend
npm install
npm run dev
```
Open the printed local URL (usually `http://localhost:5173`).

### 2. Backend (optional — only needed if you want the dataset served over HTTP instead of generated client-side)
```bash
cd backend
npm install
npm run start
```
Runs on `http://localhost:4000`. Endpoints: `GET /api/locations`, `GET /api/samples`, `GET /api/samples/:locationId`.

The frontend works fully **without** the backend running — it generates the same dataset locally. If you start the backend, flip `USE_BACKEND = true` at the top of `frontend/src/data/generateData.js` to fetch from it instead.

## What's implemented and working
- Landing page with hero, animated background, feature cards → routes into dashboard
- Campus overview cards with animated circular gauges (health score, risk level, active alerts, samples collected, weekly trend)
- Interactive campus map (your real image) — every building is a clickable hotspot, color-coded green/yellow/red, opens a detail panel with live pH / viral load / bacterial load / turbidity / last sample time / risk score / recommendation
- 7-day AI outbreak prediction chart with confidence % and plain-English reasons
- Disease probability panel (COVID / Norovirus / Typhoid / Cholera / Influenza) as circular progress rings, computed from sensor values
- Explainable-AI alert card — every red/yellow alert lists the exact reasons and a confidence score
- Environmental Health Index gauge (0–100) with contributing factors
- Historical timeline slider (30 days) that re-renders every chart/map/alert for the selected day
- Weather correlation panel + rain → overflow → contamination → risk causal chain
- Smart sampling route (optimized visiting order across real buildings)
- AI Health Assistant chatbot — rule-based, answers from live dashboard state (e.g. "Why is B14 red?", "Which building is safest?")
- Automated emergency response plan generator for red alerts, downloadable as a text report
- Notification center with animated live-feed cards
- Live simulation switcher: Normal Week / Festival Week / Heavy Rain / Winter / Summer / Hostel Outbreak — instantly recomputes every chart, alert, map color and prediction
- Privacy dashboard explaining the anonymous, building-level (not individual) monitoring model
- Analytics: line, area, pie, bar, and radar charts, all animated
- Admin panel: Generate Report, Export CSV (real file download), Export PDF (print-formatted), Run Simulation, Reset Data — all functional
- Deployment view: last updated, sampling frequency, coverage, accuracy, system status
- Dark theme by default with a light-mode toggle; fully responsive

## Design
Green / cyan / amber accents on a deep slate background — a public-health SaaS look (think a hybrid of a hospital ops dashboard and a modern analytics product), deliberately **not** a neon cyberpunk theme.
